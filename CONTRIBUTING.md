## Getting Started

1. **Fork the Repository**
   ```bash
   # Fork the repo on GitHub, then clone your fork
   git clone https://github.com/your-username/portfolio.git
   cd portfolio
   ```

2. **Create a Branch**
   ```bash
   git checkout -b feature/your-feature-name
   # or
   git checkout -b fix/issue-description
   ```

3. **Make Your Changes**
   - Edit the files as needed
   - Test your changes thoroughly
   - Follow the style guidelines

4. **Commit Your Changes**
   ```bash
   git add .
   git commit -m "Add: brief description of changes"
   ```

5. **Push to Your Fork**
   ```bash
   git push origin feature/your-feature-name
   ```

6. **Create a Pull Request**
   - Go to the original repository on GitHub
   - Click "New Pull Request"
   - Select your branch and describe your changes

## Development Setup

### Prerequisites
- A modern web browser (Chrome, Firefox, Safari, Edge)
- Text editor or IDE (VS Code, Sublime Text, etc.)
- Git for version control

### Local Development
1. Clone the repository
2. Open `index.html` in your browser
3. Make changes to HTML, CSS, or JavaScript files
4. Refresh the browser to see changes

## Submitting Changes

### Pull Request Process

1. **Ensure Quality**
   - Test your changes across different browsers
   - Check for responsive design on mobile devices
   - Validate HTML and CSS
   - Ensure JavaScript works without errors

2. **Update Documentation**
   - Update README.md if needed
   - Add comments to complex code
   - Include examples for new features

3. **Write a Good PR Description**
   - Explain what changes you made
   - Reference any related issues
   - Include screenshots for UI changes
   - List any breaking changes

4. **PR Review Process**
   - Maintainers will review your PR
   - Address any feedback promptly
   - Make requested changes
   - Once approved, your PR will be merged

### Commit Message Guidelines

Use clear and descriptive commit messages:

```
Type: Brief description

Detailed explanation if needed

Types:
- Add: New feature or functionality
- Fix: Bug fix
- Update: Changes to existing features
- Remove: Removed feature or code
- Style: Code style changes (formatting, etc.)
- Docs: Documentation changes
- Test: Adding or modifying tests
```

Examples:
```
Add: Contact form validation
Fix: Mobile navigation toggle issue
Update: Hero section animations
Style: Format CSS for better readability
```

## Style Guidelines

### HTML
- Use semantic HTML5 elements
- Include proper meta tags
- Use descriptive class names
- Ensure accessibility (alt tags, ARIA labels)

### CSS
- Use CSS custom properties (variables)
- Follow BEM methodology for class naming
- Write responsive CSS (mobile-first)
- Use consistent spacing and formatting
- Comment complex styles

### JavaScript
- Use modern ES6+ features
- Write clean, readable code
- Add comments for complex logic
- Handle errors gracefully
- Use consistent naming conventions

### General
- Keep code DRY (Don't Repeat Yourself)
- Use meaningful variable and function names
- Format code consistently
- Remove unused code and comments

## Reporting Issues

When reporting issues, please include:

### Bug Reports
- **Title**: Clear, descriptive summary
- **Description**: Detailed explanation of the problem
- **Steps to Reproduce**: Step-by-step instructions
- **Expected Behavior**: What should happen
- **Actual Behavior**: What actually happens
- **Browser/Device**: Your browser and device info
- **Screenshots**: If applicable

### Issue Template
```markdown
## Bug Report

**Description:**
A clear description of the bug.

**Steps to Reproduce:**
1. Go to '...'
2. Click on '...'
3. See error

**Expected Behavior:**
What you expected to happen.

**Actual Behavior:**
What actually happened.

**Environment:**
- Browser: [e.g., Chrome 91]
- Device: [e.g., iPhone 12, Windows 10]
- Screen Size: [e.g., 1920x1080]

**Screenshots:**
If applicable, add screenshots.
```
